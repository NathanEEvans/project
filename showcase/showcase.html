Explorer Views

There are three explorer panes: Projects, Files and Services.

Project Explorer

The Project Explorer presents the projects in a structured layout, where the (test) sources, (test) resources, webapp files, project and maven settings are conventiently grouped. Saves you a few clicks.

It supports a wide variety of files (the little green dots are the versioning labels green/blue/red)

Trough the Project context menu you can run maven commands on the project, run git commands and right click deploy it into the Tomcat container.

Projects can be closed for later use, well, yes and opened again

File Explorer

The File Explorer gives a plain old filesystem view. Mainly to make sure you can access all your files at any time when they, for example, do not show up in the Project explorer. We can only handle a fixed variation of expected layouts.

Service Explorer

The Services Explorer enables you to browse and control the 'supporting acts', like Databases, Web Services, Tomcat server(s) ... well, take a look at the snapshot here.

The Tomcat Service can be explorer to see what is deployed

The Maven repositories can be browsed, readonly opened and files downloaded.

View a pom file in the repository for instance

You can create Database connections to browse (work in progress) their contents, create new databases and edit tables and data
